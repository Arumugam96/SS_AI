. Kubernetes Architecture and Core Components

cluster 
Master Node Components:

API Server (kube-apiserver): Central control plane component that exposes the Kubernetes API.
 It's the entry point for all administrative commands and interactions.

Etcd: A consistent and highly-available key-value store used for persistent storage of all cluster data.

Controller Manager (kube-controller-manager): Manages various controllers that handle routine tasks such as scaling, node management, and replication.

Scheduler (kube-scheduler): Assigns workloads (Pods) to nodes based on resource availability, affinity/anti-affinity rules, and other factors.

Worker Node Components:

Kubelet: An agent that runs on each node, ensuring that containers are running as expected in a Pod.

Kube-proxy: Manages networking rules on nodes, ensuring that networking traffic is routed correctly.

Pods: The smallest deployable units in Kubernetes, consisting of one or more containers that share storage, network, and namespace.
